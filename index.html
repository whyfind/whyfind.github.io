web前端题目---面试者的福利

发表于 2016-02-19
1.js有哪些数据类型呢？

答：[String, Number, Undefined, Boolean, Null, Object, Function, Array]

2.这些数据类型，哪些是引用类型的呢？

答：[Object, Function, Array]

3.说说函数表达式和函数声明的区别？

函数表达式：var func = function () {}

函数声明： function func () {}

两者的区别：当使用函数声明的形式来定义函数时，可以在声明之前被调用。函数表达式的形式来定义函数则不可以。会报错的。

4.闭包是什么？优缺点？

闭包的实质是一个函数，是一个用于返回局部变量值的函数，因为在全局中，受JavaScript链式作用域结构的影响，父级变量中无法访问到子级的变量值，为了解决这个问题，才使用闭包这个概念。由于闭包时，变量的值都保存到内存中，会导致页面加载时内存消耗很大，IE会导致内在泄露，因此尽量少用或用时要及时删除变量。

5.splice()的用法

index:起始位置；
howmany：要删除的长度；
item1,…,itemX:添加的项；
splice(index,howmany,item1,…,itemX)

var a =[1,2,3]; 
a.splice(1,1,2,3,4); 
console.log(a);//[1, 2, 3, 4, 3]
6.解释下浮动和它的工作原理？清除浮动的技巧？

浮动元素脱离文档流，不占据空间。浮动元素碰到包含它的边框或者浮动元素的边框停留。

  1. 使用空标签清除浮动。这种方法是在所有浮动标签后面添加一个空标签 定义css clear:both. 弊端就是增加了无意义标签。
  2. 使用overflow。给包含浮动元素的父标签添加css属性 overflow:auto; zoom:1; zoom:1用于兼容IE6。
  3. 使用after伪对象清除浮动。该方法只适用于非IE浏览器。具体写法可参照以下示例。使用中需注意以下几点。一、该方法中必须为需要清除浮动元素的伪对象中设置 height:0，否则该元素会比实际高出若干像素；
7.详细解释ajax的工作原理？
  1. 创建ajax对象(XmlHttpRwquest/ActiveXObject(Microsoft.XMLHttp))
  2. 判断数据传输方式(get/post)
  3. 打开链接open()
  4. 发送send()

当ajax完成第四步(onreadystatechange)数据接收完成，判断http响应状态，如果为200-300之间或者304(缓存)，执行回掉函数

8.innerHTML，innerText和outerHTML，outerText的区别？

举个例子来说吧。

<div><span>内容</span></div>

使用这几个来获取上面div的内容的话，区别如下：

innerHTML:"<span>内容</span>",带有html标签

innerText:"内容",不带html标签

outerHTML:"<div><span>内容</span></div>"

outerText:获取元素跟innterText是一样的。

apply-call-bind的用法以及他们的区别

发表于 2016-02-17

一、call、apply用法

1.1、For Example
var one = {
      content : "结束吧！",
      year : "2015年是去年了。",
      print : function() {
        console.log(this.content + this.year );
      }
}

one.print();//结束吧！2015年是去年了。

这样one.print()的打印结果是显而易见的不用多说。

var two = {
      content : "时间很快~",
      year : "今年是2016年了。",
}

现在我们新增加一个 two对象。那么需求来了：two对象本身没有print方法，那么如何用one对象里面的print方法来显示数据呢？那就可以用到call或者apply了。

one.print.call(two);  //时间很快~今年是2016年了。
one.print.apply(two); //时间很快~今年是2016年了。
1.2 应用场景

1.2.1 数组之间的追加

var arr1 = [1 , 2 , 3 , 4]; 
var arr2 = [5 , 6 , "name"]; 
Array.prototype.push.apply(arr1, arr2); 
console.log(arr1);//[1, 2, 3, 4, 5, 6, "name"]

1.2.2 获取数组中的最大值和最小值

var arr3 = [5, 4 , 120 , -230 ]; 
var maxValue = Math.max.apply(Math, arr3),   //120
    maxValue = Math.max.call(Math,5, 4 , 120 , -230); //230

数组本身没有max方法。但是Math 有，我们就可以借助 call 或者 apply 使用其方法。

二 、call和apply之间的区别

说到这两者之间的区别，就得先看看他们的定义了。你就不难发现其中的区别。

2.1 call方法 

调用一个对象的一个方法，以另一个对象替换当前对象。 

call([thisObj[,arg1[, arg2[, [,.argN]]]]]) 

参数：

thisObj:可选项。将被用作当前对象的对象。 

arg1, arg2, , argN:可选项。将被传递方法参数序列。

2.2 apply方法 

应用某一对象的一个方法，用另一个对象替换当前对象。

apply([thisObj[,argArray]]) 

参数： 

thisObj:可选项。将被用作当前对象的对象。

argArray:可选项。将被传递给该函数的参数数组。

2.3 For Example 

var func = function(arg1, arg2) {};
func.call(this, arg1, arg2);
func.apply(this, [arg1, arg2]);

其中 this 是你想指定的上下文，他可以是任何一个 JavaScript 对象(JavaScript 中一切皆对象)，call 需要把参数按顺序传递进去，而 apply 则是把参数放在数组里。

可以看出两者的区别：对于 apply、call 二者而言，作用完全一样，只是接受参数的方式不太一样。

三、bind的用法

For Example

var func= function(){
    console.log(this.name);
}
var foo = {
    "name" : "whyFind"
}
var usebind = func.bind(foo);
usebind(); //whyFind

MDN的解释是：bind()方法会创建一个新函数，称为绑定函数，当调用这个绑定函数时，绑定函数会以创建它时传入 bind()方法的第一个参数作为 this，传入 bind() 方法的第二个以及以后的参数加上绑定函数运行时本身的参数按照顺序作为原函数的参数来调用原函数。

在一道非常经典的面试题中就可以用bind来解决实际问题：

var foo = function(){
    this.myName = "Hello, World!";
}
foo.prototype.sayHello = function(){
    console.log(this.myName);
}
foo.prototype.bar = function(){
    setTimeout(this.sayHello, 1000);
}
var f = new foo;
f.bar(); //undefined

这里的myNmae并不能被正确打印出来。那么怎么办呢？看这里。。

var foo = function(){
    this.myName = "Hello, World!";
}
foo.prototype.sayHello = function(){
    console.log(this.myName);
}
foo.prototype.bar = function(){
    setTimeout(this.sayHello.bind(this), 1000);
}
var f = new foo;
f.bar(); //Hello, World!

这样就能正确的打印出来了。

四、总结

1.apply 、 call 用法功能大致相同，只是传入的参数的方法不一样；

2.apply 、 call 、bind 三者都是用来改变函数的this对象的指向的；

3.apply 、 call 、bind 三者第一个参数都是this要指向的对象，也就是想指定的上下文；

4.apply 、call 则是直接返回结果(立即调用)；bind 是返回对应函数体，稍后应该再调用一次。
